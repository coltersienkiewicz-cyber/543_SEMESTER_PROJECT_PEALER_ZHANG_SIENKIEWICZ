# Crop Diversity as an Insurance Risk Metric
### Does geographic crop diversity predict agricultural insurance loss ratios?

This project uses USDA Risk Management Agency (RMA) data to test whether **Shannon crop diversity** — a measure of how many different crops are grown in a county and how evenly distributed they are — is a meaningful predictor of insurance risk. The analysis spans county-year observations from 1989–2024 across three RMA datasets, building toward a hypothesis that more diverse counties are more resilient to crop losses.

---

## Research Hypothesis

> Counties with higher crop diversity (as measured by Shannon entropy) will exhibit lower insurance loss ratios and lower claim volatility than monoculture-dominant counties, after controlling for exposure size and coverage type.

## Pipeline — Run Order

Run scripts in this order. Each script reads from `data/` and writes back to `data/` or `graphics/`.

### Step 0 — Download raw data
```bash
python read_crop_loss_api.py
```
Downloads pipe-delimited `.txt` files from the RMA website for years 1989–2024 into `crop_loss_data/`. This may take several minutes. Some early years (pre-1999) have limited coverage for certain datasets — that is expected.

### Step 1 — Parse raw files to CSV
```bash
python 01_load_data.py
```
Reads all `.txt` files from the three subdirectories in `crop_loss_data/`, assigns column names from the RMA data dictionaries, and saves three CSVs to `data/`:
- `colsommonth.csv` — cause-of-loss by month
- `sobsccc.csv` — state/county/crop policy summaries
- `sobtpu.csv` — type/practice/unit-structure detail

### Step 2 — Build diversity metrics
```bash
python 02_build_diversity.py
```
Computes five diversity metrics per county-year from insured-acreage crop shares (Acres rows only):

| Metric | Description |
|---|---|
| `shannon` | Shannon entropy H' = −Σ pᵢ ln(pᵢ) |
| `simpson` | Gini-Simpson D = 1 − Σ pᵢ² |
| `evenness` | Pielou's J = H' / ln(S) |
| `dominance` | Berger-Parker d = max(pᵢ) |
| `n_crops` | Species richness (count of distinct crops) |

Also computes `shannon_stability` (std of H' across years per county) and merges county landmass data.

**Outputs:** `data/county_year_df.csv`, `data/county_summary.csv`

> **Requires:** `county_landmass.csv` in the project root — see Data Sources below.

### Step 3 — Cluster analysis + Random Forest
```bash
python 04_cluster_analysis.py
```
Runs K-Means clustering across all diversity features. Optimal k is chosen by silhouette score. Clusters are ordered and labeled by mean Shannon value (Cluster 1 = least diverse, Cluster N = most diverse). Also runs a Random Forest regressor predicting `mean_indem_acre` and plots feature importances.

**Outputs:** updates `data/county_summary.csv` with `diversity_category` column, plots to screen.

### Step 4 — Volatility regression
```bash
python 07_volatility_regression.py
```
OLS regression of five diversity metrics on `cv_indemnity` (coefficient of variation of indemnity/acre across years). Reports VIF, train/test R², RMSE, and a coefficient plot.

### Step 5 — Cause-of-loss by cluster
```bash
python 08_cause_of_loss_by_cluster.py
```
For each combination of diversity cluster × cause-of-loss group (drought, heat, cold, wind/storm, precipitation, pest, disease, fire), computes median loss intensity (indemnity/acre) and median claim volatility (CV). Runs Kruskal-Wallis tests to determine whether cluster significantly predicts volatility within each cause group.

**Requires:** `04_cluster_analysis.py` to have been run first (needs `diversity_category` in `county_summary.csv`).

### Step 6 — Build county-year panel
```bash
python 09_county_panel.py
```
Assembles one row per county-year combining all three source tables plus cause-of-loss breakdowns (indemnity and policy counts per cause group, event flags). Computes `loss_ratio` bottom-up from premium/indemnity components rather than averaging pre-computed row-level ratios.

**Output:** `data/county_panel.csv`

### Step 7 — Panel regressions
```bash
python 10_panel_regression.py
```
Two OLS regressions with 75/25 train/test splits:
1. `indemnity_per_acre` ~ diversity + exposure features (county-year level, ~10k+ obs)
2. `cv_indemnity` ~ diversity + exposure features (county level, ~2.6k obs)

Produces a correlation matrix, coefficient plots, and a side-by-side coefficient comparison between the two models.

### Step 8 — Mega correlation matrix
```bash
python 11_mega_correlation.py
```
Builds a ~50-variable Spearman correlation matrix spanning diversity metrics, loss outcomes, cause-of-loss breakdowns, exposure features, and policy features. Axes are reordered by hierarchical clustering and color-coded by variable group.

**Output:** `graphics/mega_corr_matrix.png`

### Step 9 — Feature matrix
```bash
python 12_feature_matrix.py
```
Produces the clean, analysis-ready feature matrix used by EDA and ML scripts:

| Column | Description |
|---|---|
| `shannon_diversity` | Shannon H' |
| `num_unique_crops` | Distinct insured commodities |
| `top_crop_share` | Berger-Parker dominance |
| `evenness` | Pielou's J |
| `total_liability` | Total insured liability ($) |
| `avg_coverage_level` | Liability-weighted mean coverage level |
| `dominant_insurance_plan` | Plan with most liability in county-year |
| `num_loss_causes` | Distinct cause-type groups with claims |
| `weather_loss_share` | Fraction of indemnity from weather causes |
| `loss_ratio` | **Target:** indemnity / premium |

**Output:** `data/feature_matrix.csv`

### Step 10 — Exploratory data analysis
```bash
python 13_eda.py
```
Produces five figures in `eda_plots/`:
1. Scatter: Shannon diversity vs. loss ratio, colored by state (with OLS trend line)
2. Boxplot: loss ratio distributions by diversity quartile
3. Heatmap: feature correlation matrix
4. Side-by-side choropleth map: avg diversity vs. avg loss ratio
5. Time series: county-median diversity and loss ratio by year

> **Requires:** a county shapefile at `Farm Resource Regions/tl_2025_us_county/tl_2025_us_county.shp` — see Data Sources below.

### Step 11 — Hypothesis tests
```bash
python 14_hypothesis_tests.py
```
Formal statistical tests of the core hypothesis:
- **Spearman correlation** between Shannon diversity and loss ratio
- **Kruskal-Wallis test** comparing loss ratios across diversity quartiles
- **Mann-Whitney U test** (high vs. low diversity counties)

### Step 12 — ML modeling + SHAP
```bash
python 15_modeling.py
```
Gradient Boosting regressor predicting `loss_ratio` from all features in the feature matrix. Reports train/test R² and RMSE, plots SHAP values to quantify how much diversity contributes relative to other predictors.

## Data Sources

### RMA Crop Insurance Data
Downloaded automatically by `read_crop_loss_api.py` from the USDA RMA website. Three dataset types are used:

| Dataset | RMA Name | Description |
|---|---|---|
| `colsommonth` | Cost of Loss | Monthly indemnities by cause-of-loss code |
| `sobsccc` | State/County/Crop | Policy counts, premiums, liability by commodity |
| `sobtpu` | Type/Practice/Unit | Insured acreage by crop type and practice |

### County Landmass (`county_landmass.csv`)
Required for computing `pct_land_insured`. This file must contain:
- `FIPS_state` — 2-digit state FIPS code
- `FIPS_county` — 3-digit county FIPS code
- `land_sq_mi` — county land area in square miles

Available from the [US Census Bureau TIGER/Line files](https://www.census.gov/geographies/mapping-files/time-series/geo/tiger-line-file.html) or the [USDA ERS county typology dataset](https://www.ers.usda.gov/data-products/county-typology-codes/).

### County Shapefile (for choropleth maps)
`13_eda.py` expects a shapefile at:
```
Farm Resource Regions/tl_2025_us_county/tl_2025_us_county.shp
```
Download from the [Census Bureau TIGER/Line Shapefiles](https://www.census.gov/cgi-bin/geo/shapefiles/index.php) (Counties, 2025 vintage).

## Key Design Decisions

**Acres-only filtering for diversity:** Diversity metrics are computed only from rows where `reporting_level_type == 'Acres'`. Rows measured in Trees, Head, or Tons are excluded so that crop proportions reflect actual land-use shares and can be meaningfully compared across counties.

**Loss ratio construction:** Rather than averaging the pre-computed `loss_ratio` column (which is unit-weighted), `09_county_panel.py` reconstructs loss ratio bottom-up as `sum(indemnity) / sum(premium)` from the `sobsccc` table, which has the cleanest premium denominator.

**Outlier handling:** The top 1% of `loss_ratio` and `cv_indemnity` values are clipped in regression and EDA scripts to reduce distortion from audit adjustments and carryover payments (which can produce negative or very large ratios).

**Dixie County exclusion:** Dixie County, FL is dropped from `county_summary.csv` as a structural outlier in the indemnity/acre distribution.
